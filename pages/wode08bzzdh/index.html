本站自动化

# 修改代码触发自动部署

本站有两种部署方式，一种是手动执行deploy.sh，第二种是用GitHub Action自动化持续集成，第二种更方便也更自动化，所以只介绍第二种喽！

本站根据VuePress和vdoing的推荐，接入了GitHub Action，支持修改完代码提交之后，触发GitHub Action，自动build和deploy发布资源，同时也是因为它，我们可以在github直接新增或者修改文章，修改完成之后，会自动触发build和deploy，不需要我们手动构建和发布，自动部署。

具体配置的话VuePress和vdoing都有详细文档，这里就不作介绍了，贴链接如下：

  • VuePress GitHub CI (opens new window)
  • vdoing GitHub CI (opens new window)

# 可能的问题

这里记录一下过程中可能碰到的问题。

# GitHub Action build no Permission

报错信息如下：

remote: Permission to 你的仓库地址 denied to github-actions[bot].
fatal: unable to access '你的仓库地址': The requested URL returned error: 403
Error: Process completed with exit code 128.
1
2
3

因为.github/workflows/ci.yml文件配置需要取你的secrets.ACCESS_TOKEN去做自动化，这个token需要配置权限的，报错之后，可以去用户设置（不是当前仓库的设置）去修改权限，具体设置地址：Settings/Developer Settings/Personal access tokens/Fine-grained personal access tokens，点进去之前设置的token，编辑权限为当前仓库或所有仓库，以及下面的具体权限更改为读和写就好了。
